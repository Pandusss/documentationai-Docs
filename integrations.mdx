## Setting Up Webhooks

Webhooks enable real-time automation. You receive payloads from Doulot Doulottt when events like page updates occur.

### Register a Webhook

Follow these steps to create a webhook endpoint.

<Steps>
  <Step title="Navigate to Integrations" icon="settings">
    Go to your workspace settings > Integrations > Webhooks.
  </Step>
  <Step title="Add Endpoint" icon="plus">
    Enter your webhook URL, such as `https://your-webhook-url.com/doulot`.
  </Step>
  <Step title="Select Events" icon="zap">
    Choose events like `page.updated` or `comment.created`.
  </Step>
  <Step title="Save and Test" icon="check-circle">
    Save the webhook and trigger a test event.
  </Step>
</Steps>

### Webhook Payload Example

Use this API call to register programmatically.

<ParamField path="webhookUrl" param-type="string" required="true">
  Your public endpoint URL.
</ParamField>

<ParamField path="events" param-type="array" required="true">
  Array of event names, e.g., `["page.updated", "page.created"]`.
</ParamField>

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X POST https://api.example.com/v1/webhooks \\
    -H "Authorization: Bearer YOUR_API_KEY" \\
    -H "Content-Type: application/json" \\
    -d '{
      "webhookUrl": "https://your-webhook-url.com/doulot",
      "events": ["page.updated", "comment.created"]
    }'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/webhooks', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      webhookUrl: 'https://your-webhook-url.com/doulot',
      events: ['page.updated', 'comment.created']
    })
  });
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "id": "wh_1234567890",
    "webhookUrl": "https://your-webhook-url.com/doulot",
    "events": ["page.updated", "comment.created"],
    "active": true
  }
  ```
</Response>

## Best Practices

<Callout kind="tip">
  Always validate incoming webhook payloads using HMAC signatures to prevent unauthorized access.
</Callout>

- Use unique secrets for each webhook.
- Implement retry logic for failed deliveries.
- Monitor webhook logs in your dashboard.

<Expandable title="Advanced: Custom Headers" default-open="false">
  Add custom headers during registration:
  
  ```json
  {
    "headers": {
      "X-Custom-ID": "your-app-id"
    }
  }
  ```
</Expandable>
